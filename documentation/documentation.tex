\chapter*{Rezumat}
\addcontentsline{toc}{chapter}{Rezumat}
\markboth{Rezumat}{Rezumat}

Prezenta lucrare de licență descrie proiectarea, implementarea și testarea unei aplicații mobile
cross-platform denumită \textbf{FocusQuest}, dezvoltată cu React Native și TypeScript.
Aplicația îmbină elemente de joc de rol (RPG) cu instrumente de productivitate, fiind
destinată oricui este interesat de productivitate, studiu și jocuri inteligente cu AI mai,
ales elevilor care urmează să susțină admiterea pentru intrarea la Universitatea Politehnica Timișoara.

Principalele contribuții ale lucrării sunt:
\begin{itemize}[leftmargin=1.5cm]
  \item Integrarea unui model de rețea neuronală convoluțională (\textit{ResNet-50 INT8}) rulat
    \textit{on-device} prin ONNX Runtime React Native, utilizat pentru clasificarea imaginilor
    în jurnalul digital al utilizatorului.
  \item Integrarea motorului de șah \textit{Maia Chess} — un model antrenat să imite stilul
    de joc uman la diverse niveluri Elo (1100–1500) — rulat complet local, fără conexiune la
    internet.
  \item Un sistem de economie virtuală bazat pe monede câștigate prin sesiuni de focalizare,
    jocuri și activități de învățare, utilizat pentru deblocarea conținutului de dificultate
    mai ridicată.
  \item Un modul de învățare structurat pe trei discipline matematice (algebră, analiză şi
    trigonometrie), cu probleme matematice şi soluții pas-cu-pas
    randate cu KaTeX.
  \item Persistența stării aplicației folosind AsyncStorage și un gestionar global de stare
    bazat pe React Context API cu \texttt{useReducer}.
\end{itemize}

Aplicația a fost testată pe dispozitive Android și respectă arhitectura componentelor React
Native, cu o separare clară a logicii de afaceri față de stratul de prezentare.

\textbf{Cuvinte cheie:} React Native, TypeScript, ONNX Runtime, ResNet-50, Maia Chess,
gamificare, aplicație mobilă, productivitate, algoritmi de backtracking, KaTeX.

\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}
\markboth{Abstract}{Abstract}

This bachelor thesis describes the design, implementation and testing of a cross-platform
mobile application called \textbf{FocusQuest}, built with React Native and TypeScript.
The application combines role-playing game (RPG) mechanics with productivity tools, aimed at
anyone interested in productivity, learning and intelligent games.

The main contributions are:
\begin{itemize}[leftmargin=1.5cm]
  \item Integration of a convolutional neural network (\textit{ResNet-50 INT8}) running
    \textit{on-device} via ONNX Runtime React Native, used to classify images in the user's
    digital journal.
  \item Integration of the \textit{Maia Chess} engine — a model trained to mimic human chess
    playing style at various Elo levels (1100–1500) — running fully locally, without an
    internet connection.
  \item A virtual economy system based on coins earned through focus sessions, games and
    learning activities, used to unlock higher-difficulty content.
  \item A learning module structured around three mathematical disciplines (algebra, analysis
    and trigonometry), with mathematical problems and step-by-step
    solutions rendered with KaTeX.
  \item Application state persistence using AsyncStorage and a global state manager based on
    the React Context API with \texttt{useReducer}.
\end{itemize}

\textbf{Keywords:} React Native, TypeScript, ONNX Runtime, ResNet-50, Maia Chess,
gamification, mobile application, productivity, backtracking algorithms, KaTeX.

\chapter{Introducere}

\section{Motivație și context}

Accesul tinerilor la instrumente moderne de studiu s-a diversificat semnificativ în ultimul
deceniu. Telefoanele inteligente sunt omniprezente în rândul elevilor și studenților, iar
aplicațiile mobile dedicate productivității și educației reprezintă un segment în creștere
rapidă al pieței de software. Cu toate acestea, există o lipsă acută de aplicații care să
combine în mod integrat:
\begin{enumerate}
  \item Tehnici de productivitate validate din punct de vedere \textit{\c{s}tiin\c{t}ific} (ex. tehnica Pomodoro);
  \item Conținut de pregătire pentru examene specifice sistemului de învățământ românesc;
  \item Mecanisme de gamificare care să mențină motivația pe termen lung;
  \item Modele de inteligență artificială care să funcționeze \textit{offline}, pe dispozitiv.
\end{enumerate}

\textbf{FocusQuest} răspunde acestei nevoi prin crearea unui ecosistem mobil coerent în care
studiul și distracția coexistă. Aplicația este construită cu un design vizual retro-neon RPG,
conceput să fie atractiv pentru generația Z, obișnuită cu estetica jocurilor video indie.

\section{Obiectivele lucrării}

Obiectivele principale ale acestei lucrări de licență sunt:
\begin{itemize}
  \item Proiectarea și implementarea unei aplicații mobile React Native complete, cu navigare
    multi-ecran, gestionare globală a stării și persistență locală a datelor.
  \item Integrarea a două modele ONNX \textit{on-device}: ResNet-50 pentru clasificarea
    imaginilor și Maia Chess pentru jocul de șah cu inteligență artificială.
  \item Construirea unui sistem de gamificare funcțional cu monede, realizări deblocabile,
    inventar și dificultăți progresive.
  \item Furnizarea unui modul educațional cu exerciții matematice structurate, complet
    cu răspunsuri multiple şi soluții detaliate randate cu KaTeX.
  \item Demonstrarea viabilității rulării modelelor de rețele neuronale direct pe dispozitive
    mobile cu resurse limitate.
\end{itemize}

\section{Structura lucrării}

Lucrarea este organizată în nouă capitole:
\begin{itemize}
  \item \textbf{Capitolul 2} prezintă stadiul actual al domeniului (State of the Art),
    analizând aplicații similare existente și comparând funcționalitățile acestora cu
    FocusQuest.
  \item \textbf{Capitolul 3} descrie tehnologiile și instrumentele utilizate în dezvoltare.
  \item \textbf{Capitolul 4} prezintă arhitectura generală a aplicației.
  \item \textbf{Capitolul 5} detaliază sistemul de gestiune a stării și persistența datelor.
  \item \textbf{Capitolul 6} descrie modulele funcționale principale ale aplicației.
  \item \textbf{Capitolul 7} analizează integrarea modelelor de inteligență artificială.
  \item \textbf{Capitolul 8} prezintă testarea și evaluarea aplicației.
  \item \textbf{Capitolul 9} concluzionează lucrarea și propune direcții viitoare de
    dezvoltare.
\end{itemize}

\chapter{Stadiul actual al domeniului (State of the Art)}

\section{Aplicații de productivitate și focalizare}

\subsection{Forest – Stay Focused}

Forest este una dintre cele mai cunoscute aplicații de productivitate cu element de
gamificare. Utilizatorul plantează un copac virtual care crește cât timp nu folosește
telefonul; dacă iese din aplicație, copacul moare. Aplicația colaborează cu organizații
de plantare a copacilor reali, ceea ce adaugă o dimensiune de impact social.

\textbf{Comparație cu FocusQuest:} Forest se concentrează exclusiv pe tehnica Pomodoro
și nu integrează niciun conținut educațional, niciun joc inteligent și niciun model AI.
FocusQuest extinde conceptul de focalizare cu un ecosistem complet: sesiunile de focus
generează monede care permit accesul la jocuri și conținut educațional.

\subsection{Todoist și Notion}

Todoist și Notion sunt instrumente de gestionare a sarcinilor și note-taking, utilizate
frecvent de studenți pentru organizarea studiului. Deși extrem de versatile, aceste aplicații
nu oferă gamificare intrinsecă, nu integrează modele AI locale şi nu sunt orientate spre
practică structurată de exerciții matematice.

\subsection{Flora – Green Focus}

Flora urmează un model similar cu Forest, cu accent pe productivitate în grup și provocări
sociale. Nu include elemente RPG, nu are jocuri integrate și nu are funcționalitate offline
bazată pe AI.

\section{Aplicații de învățare și pregătire pentru examene}

\subsection{Khan Academy}

Khan Academy oferă un curriculum vast, inclusiv matematică avansată, cu exerciții
interactive și progres vizibil. Platforma este gratuită și bine structurată, dar:
\begin{itemize}
  \item Nu este orientată spre conținut specific admiterii la universitățile românești;
  \item Nu are elemente de gamificare RPG;
  \item Nu integrează jocuri sau un sistem de monede;
  \item Necesită conexiune la internet.
\end{itemize}

\subsection{Brilliant.org}

Brilliant.org oferă cursuri interactive de matematică și informatică cu un nivel ridicat de
interactivitate. Interfața este modernă și angajantă, dar platforma este orientată spre
gândire conceptuală, nu spre pregătirea pentru examinări de tip grilă specifice.

\subsection{Duolingo}

Duolingo este exemplul canonic de aplicație educațională cu gamificare: streak-uri zilnice,
monede virtuale, nivele și personaje animate. FocusQuest se inspiră din principiile de design
ale Duolingo (streak-uri, monede, recompense), adaptate contextului unui RPG și extins cu
funcționalități AI locale.

\textbf{Diferența cheie față de FocusQuest:} FocusQuest combină zona de practică de exerciții
cu zona de jocuri (șah AI, Sudoku) și cu zona de productivitate (timer focus, geolocation)
într-o singură aplicație cu o economie virtuală unificată.

\section{Aplicații de șah cu AI}

\subsection{Chess.com și Lichess}

Chess.com și Lichess sunt platformele dominante de șah online. Ambele oferă oponenți AI
la multiple niveluri de dificultate. Lichess este open-source și oferă motorul Stockfish,
unul dintre cei mai puternici motori de șah din lume.

\textbf{Diferența față de FocusQuest:} FocusQuest nu utilizează Stockfish (care joacă la
nivel de campion mondial). În schimb, integrează \textbf{Maia Chess}, un model antrenat să
joace \textit{uman}, comițând greșeli tipice ale jucătorilor la niveluri Elo între 1100 și 1500.
Aceasta face experiența mult mai potrivită pentru jucători începători și intermediari.
Mai mult, FocusQuest rulează modelul complet offline, fără server.

\subsection{Magnus Trainer}

Magnus Trainer este o aplicație de antrenament la șah creată de campionul mondial Magnus
Carlsen. Oferă lecții, puzzle-uri și jocuri contra AI. Aplicația este premium și nu
integrează module de studiu sau productivitate.

\section{Aplicații cu clasificare AI a imaginilor}

\subsection{Google Lens}

Google Lens oferă recunoaștere de imagini de înaltă calitate, rulând pe serverele Google.
Necesită conexiune la internet și nu poate fi integrată offline într-o aplicație mobilă.

\subsection{Microsoft Azure Custom Vision}

Serviciu cloud pentru clasificare de imagini custom. Necesită abonament și conexiune la
internet; nu este potrivit pentru scenarii complet offline.

\textbf{Diferența față de FocusQuest:} FocusQuest rulează ResNet-50 INT8 direct pe
dispozitivul utilizatorului prin ONNX Runtime, fără niciun server extern, asigurând
confidențialitatea datelor și funcționalitate offline completă.

\section{Aplicații RPG educaționale}

\subsection{Habitica}

Habitica transformă gestionarea obiceiurilor și sarcinilor zilnice într-un RPG: utilizatorul
are un personaj, câștigă experiență și obiecte, luptă cu monștri în grupuri. Este cea mai
apropiată aplicație conceptual de FocusQuest, dar:
\begin{itemize}
  \item Nu integrează modele AI locale;
  \item Nu oferă jocuri inteligente (șah, Sudoku);
  \item Nu are conținut educațional structurat cu exerciții specifice;
  \item Nu are un timer de focalizare dedicat cu animații de personaj.
\end{itemize}

\subsection{ClassDojo}

ClassDojo este orientat spre sala de clasă și gestionat de profesori, nu de elevi în mod
autonom. Nu are elemente de jocuri inteligente sau AI.

\chapter{Tehnologii și instrumente utilizate}

\section{React Native}

React Native este un framework open-source dezvoltat de Meta (Facebook) care permite
crearea de aplicații mobile native pentru iOS și Android folosind JavaScript și TypeScript.
Spre deosebire de abordările hibride (ex. Ionic, Cordova), React Native compilează
componentele la UI nativ al platformei, nu la WebView, rezultând o performanță apropiată
de aplicațiile native.

Versiunea utilizată în proiect este \textbf{React Native 0.82.0} împreună cu
\textbf{React 19.1.1}.

Avantajele care au determinat alegerea React Native:
\begin{itemize}
  \item Baza de cod partajată pentru iOS și Android;
  \item Ecosistem bogat de biblioteci (navigare, stocare, imagini, sunet);
  \item Suport nativ pentru TypeScript;
  \item Compatibilitate cu ONNX Runtime prin pachetul \texttt{onnxruntime-react-native}.
\end{itemize}

\section{TypeScript}

TypeScript este un superset al JavaScript care adaugă tipizare statică. Întregul proiect
este scris în TypeScript, ceea ce oferă:
\begin{itemize}
  \item Detectarea timpurie a erorilor la compilare;
  \item Autocompletare și refactorizare mai bune în IDE;
  \item Documentare implicită a structurilor de date complexe (ex. \texttt{GameState}).
\end{itemize}

\section{React Navigation}

Navigarea în aplicație este gestionată de biblioteca \textbf{React Navigation 7.x},
utilizând două tipuri de navigatori:
\begin{itemize}
  \item \texttt{createBottomTabNavigator} — pentru cele cinci ecrane principale
    (Focus, Arcade, Learning, Journal, Profile);
  \item \texttt{createNativeStackNavigator} — pentru ecranele secundare
    (Achievements, Statistics, Inventory, ExerciseList, ExerciseDetail).
\end{itemize}

\section{AsyncStorage}

\textbf{@react-native-async-storage/async-storage} (versiunea 2.1.0) este biblioteca
de stocare cheie-valoare persistentă utilizată pentru salvarea stării aplicației între
sesiuni. Datele sunt serializate în format JSON și stocate local pe dispozitiv sub cheia
\texttt{retro-arcade-study-state-v1}.

\section{ONNX Runtime React Native}

\textbf{onnxruntime-react-native} (versiunea 1.20.0) este un pachet care expune
ONNX Runtime — un motor de inferență pentru modele de machine learning în format ONNX
— aplicațiilor React Native. Permite rularea modelelor complet local, fără server.

Funcționalitățile utilizate:
\begin{itemize}
  \item \texttt{InferenceSession.create()} — încărcarea modelului ONNX din fișier;
  \item \texttt{session.run(feeds)} — executarea inferenței cu un dicționar de tensori
    de intrare;
  \item \texttt{Tensor} — crearea tensorilor de tip \texttt{float32} și \texttt{int64}.
\end{itemize}

\section{chess.js}

\textbf{chess.js} (versiunea 1.0.0) este o bibliotecă JavaScript de logică a jocului
de șah. Oferă:
\begin{itemize}
  \item Validarea mutărilor legale;
  \item Detecția situațiilor de șah, mat, pat;
  \item Conversia pozițiilor în format FEN (Forsyth-Edwards Notation);
  \item Generarea listei complete de mutări legale în format UCI.
\end{itemize}

\section{react-native-sound}

\textbf{react-native-sound} (versiunea 0.13.0) este un modul pentru redarea fișierelor
audio locale. Este utilizat pentru serviciul de muzică de fundal (muzică lo-fi) integrat
în ecranul principal. Fișierele audio sunt stocate în directorul
\texttt{android/app/src/main/res/raw/}.

\section{react-native-image-picker}

\textbf{react-native-image-picker} (versiunea 7.2.1) permite accesarea galeriei foto
sau a camerei dispozitivului. Este utilizat în modulul jurnal (FlashcardsScreen) pentru
adăugarea imaginilor la intrările de jurnal.

\section{react-native-fs}

\textbf{react-native-fs} (versiunea 2.20.0) oferă acces la sistemul de fișiere al
dispozitivului. Este utilizat pentru copierea fișierelor de model ONNX din bundlul
de resurse Android în directorul de cache, de unde pot fi accesate de ONNX Runtime.

\section{react-native-webview}

\textbf{react-native-webview} (versiunea 13.16.1) este utilizat de componenta
\texttt{MathRenderer} pentru randarea formulelor matematice LaTeX cu biblioteca KaTeX
(încărcată din CDN sau local).

\section{KaTeX}

\textbf{KaTeX} (versiunea 0.16.11) este o bibliotecă JavaScript de randare a formulelor
matematice LaTeX, utilizată în ecranele de exerciții pentru afișarea problemelor și
soluțiilor cu formatare matematică corectă. Este încărcată într-un WebView embed.

\section{Fluxul de date al aplicației}

Fluxul de date respectă principiul unidirecțional specific arhitecturii Flux/Redux:

\begin{enumerate}
  \item Utilizatorul interacționează cu un component UI (ex. apasă butonul de Start Timer);
  \item Componentul apelează o funcție din contextul global (ex. \texttt{completeSession});
  \item Funcția dispatch-uiește un \texttt{Action} către reducer;
  \item Reducer-ul calculează noul stat pe baza acțiunii și stării curente;
  \item Noul stat este salvat în AsyncStorage;
  \item Toate componentele care consumă contextul sunt re-randate cu noul stat.
\end{enumerate}

\section{Sistemul de acțiuni (Reducer)}

Reducer-ul gestionează 20 de tipuri de acțiuni, dintre care cele mai importante sunt:

\begin{description}
  \item[\texttt{COMPLETE\_SESSION}] Finalizarea unei sesiuni de focus. Calculează monedele
    câștigate (\textit{baseReward} + \textit{streakBonus}), actualizează streak-ul și
    salvează sesiunea cu datele de geolocation.
  \item[\texttt{ADD\_FLASHCARD / UPDATE\_FLASHCARD / DELETE\_FLASHCARD}] Gestionarea
    intrărilor din jurnal.
  \item[\texttt{COMPLETE\_CHESS\_GAME / COMPLETE\_MAIA\_CHESS\_GAME}] Înregistrarea
    rezultatelor partidelor de șah cu actualizarea statisticilor și acordarea recompenselor.
  \item[\texttt{COMPLETE\_SUDOKU\_GAME}] Finalizarea unui puzzle Sudoku cu acordarea
    monedelor.
  \item[\texttt{UNLOCK\_MAIA\_DIFFICULTY / UNLOCK\_SUDOKU\_DIFFICULTY}] Deblocarea
    nivelelor de dificultate mai ridicate contra cost de monede.
  \item[\texttt{UNLOCK\_ACHIEVEMENT}] Deblocarea unei realizări.
  \item[\texttt{MARK\_EXERCISE\_SOLVED}] Marcarea unui exercițiu ca rezolvat.
  \item[\texttt{HYDRATE}] Restaurarea stării din AsyncStorage la pornirea aplicației.
\end{description}

\section{Formula de calcul a monedelor din sesiunile de focus}

Monedele câștigate per sesiune de focus sunt calculate astfel:

\begin{equation}
  \text{coinsEarned} = \max(3,\; \lfloor \text{durationMinutes} \times 1{,}5 \rfloor)
  + \left\lfloor \frac{\text{streak}}{3} \right\rfloor
  \label{eq:coins}
\end{equation}

unde \textit{durationMinutes} este durata sesiunii în minute, iar \textit{streak} este
numărul curent de zile consecutive cu cel puțin o sesiune completată.

\section{Sistemul de streak}

Streak-ul este calculat pe baza datei calendaristice (nu a orei):
\begin{itemize}
  \item Dacă ultima sesiune a fost azi: streak rămâne nemodificat;
  \item Dacă ultima sesiune a fost ieri: streak crește cu 1;
  \item Altfel: streak se resetează la 1.
\end{itemize}

\section{Persistența cu AsyncStorage}

La fiecare modificare a stării, datele sunt serializate JSON și salvate asincron:

\begin{lstlisting}[language=Java, caption={Salvarea și restaurarea stării}]
// Salvare
useEffect(() => {
  if (isHydrated) {
    AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(state));
  }
}, [state, isHydrated]);

// Restaurare la pornire
useEffect(() => {
  AsyncStorage.getItem(STORAGE_KEY).then(raw => {
    if (raw) {
      const parsed = JSON.parse(raw) as GameState;
      dispatch({ type: 'HYDRATE', payload: parsed });
    }
    setIsHydrated(true);
  });
}, []);
\end{lstlisting}

Cheia de stocare este versionată (\texttt{retro-arcade-study-state-v1}) pentru a permite
migrări viitoare de schemă fără a pierde datele existente.

hook-ul \texttt{useGame()}. Această abordare evită prop drilling și centralizează toată
logica de business.

\subsection{Jocul de Sudoku (SudokuArena)}

\subsubsection{Implementare}

Componenta \texttt{SudokuArena} implementează un joc complet de Sudoku cu:
\begin{itemize}
  \item Trei niveluri de dificultate: \textit{easy}, \textit{medium}, \textit{expert};
  \item Puzzle-uri preîncărcate din constantele \texttt{EASY\_PUZZLES},
    \texttt{MEDIUM\_PUZZLES}, \texttt{EXPERT\_PUZZLES};
  \item Un algoritm de backtracking pentru validarea și rezolvarea puzzle-urilor.
\end{itemize}

\subsubsection{Algoritmul de backtracking}

Puzzle-urile Sudoku sunt rezolvate prin backtracking recursiv. Algoritmul:
\begin{enumerate}
  \item Caută prima celulă goală (\textit{findEmpty});
  \item Încearcă numerele de la 1 la 9;
  \item Verifică validitatea plasării față de rând, coloană și sub-carou $3\times3$;
  \item Dacă nu există număr valid, revine la celula anterioară.
\end{enumerate}

\begin{lstlisting}[language=Java, caption={Algoritmul de backtracking Sudoku}]
const solveSudoku = (grid: (number | null)[][]): boolean => {
  const findEmpty = (): [number, number] | null => {
    for (let row = 0; row < 9; row++)
      for (let col = 0; col < 9; col++)
        if (grid[row][col] === null) return [row, col];
    return null;
  };

  const isValid = (num: number, row: number, col: number): boolean => {
    for (let x = 0; x < 9; x++)
      if (grid[row][x] === num || grid[x][col] === num) return false;
    const sr = Math.floor(row / 3) * 3;
    const sc = Math.floor(col / 3) * 3;
    for (let i = 0; i < 3; i++)
      for (let j = 0; j < 3; j++)
        if (grid[sr + i][sc + j] === num) return false;
    return true;
  };

  const empty = findEmpty();
  if (!empty) return true; // Rezolvat!
  const [r, c] = empty;
  for (let num = 1; num <= 9; num++) {
    if (isValid(num, r, c)) {
      grid[r][c] = num;
      if (solveSudoku(grid)) return true;
      grid[r][c] = null;
    }
  }
  return false;
};
\end{lstlisting}

\subsection{Randarea formulelor matematice (MathRenderer)}

Componenta \texttt{MathRenderer} este construită pe un \texttt{WebView} care injectează
biblioteca KaTeX pentru randarea formulelor LaTeX. Caracteristici tehnice:
\begin{itemize}
  \item Suportă atât formule inline (\texttt{\$...\$}) cât și formule bloc (\texttt{\$\$...\$\$});
  \item Definește macrouri pentru notațiile românești: \texttt{\textbackslash tg},
    \texttt{\textbackslash ctg}, \texttt{\textbackslash arctg}, \texttt{\textbackslash arcctg};
  \item Înălțimea WebView-ului se ajustează dinamic prin mesaje
    \texttt{postMessage} de la KaTeX;
  \item Nu necesită conexiune la internet dacă KaTeX este bundled.
\end{itemize}

\subsection{Fluxul de rezolvare a exercițiilor}

\begin{enumerate}
  \item Utilizatorul selectează o disciplină și un exercițiu;
  \item Problema și cele 4–5 variante de răspuns sunt afișate cu MathRenderer;
  \item Utilizatorul selectează un răspuns — imediat se afișează feedback vizual
    (verde = corect, roșu = greșit);
  \item Dacă răspunsul este corect, exercițiul este marcat ca rezolvat în
    \texttt{learning.solvedExercises};
  \item Utilizatorul poate afișa soluția pas-cu-pas independent de răspuns.
\end{enumerate}

\chapter{Integrarea modelelor de Inteligență Artificială}

\section{Arhitectura generală de inferență ONNX}

Ambele modele AI utilizate în aplicație sunt în format \textbf{ONNX (Open Neural Network
Exchange)} și sunt rulate local prin pachetul \texttt{onnxruntime-react-native}.
Acest lucru asigură:
\begin{itemize}
  \item Funcționalitate completă offline;
  \item Latență redusă (fără round-trip de rețea);
  \item Confidențialitatea datelor utilizatorului (imaginile nu sunt trimise pe servere
    externe).
\end{itemize}

Modelele sunt incluse ca resurse statice ale aplicației:
\begin{itemize}
  \item Pe \textbf{Android}: copierea din directorul \texttt{assets/models/} în
    \texttt{CachesDirectoryPath} la prima rulare (prin \texttt{react-native-fs});
  \item Pe \textbf{iOS}: accesate direct din \texttt{MainBundlePath}.
\end{itemize}

\section{Modelul ResNet-50 pentru clasificarea imaginilor}

\subsection{Descriere model}

\textbf{ResNet-50} (Residual Network cu 50 de straturi) este o arhitectură CNN
(Convolutional Neural Network) dezvoltată de Microsoft Research, câștigătoare a
competiției ImageNet Large Scale Visual Recognition Challenge (ILSVRC) 2015.

Versiunea utilizată este \textbf{ResNet-50-v1-7 quantizat INT8} din ONNX Model Zoo,
cu dimensiune redusă față de versiunea float32 originală. Modelul clasifică imaginile
în \textbf{1000 de clase ImageNet}.

Parametrii modelului:
\begin{itemize}
  \item Dimensiune intrare: $1 \times 3 \times 224 \times 224$ (batch, canale RGB, înălțime, lățime);
  \item Dimensiune ieșire: $1 \times 1000$ (logit-uri pentru 1000 de clase);
  \item Normalizare intrare: medie $\mu = [0.485, 0.456, 0.406]$, deviație standard
    $\sigma = [0.229, 0.224, 0.225]$ (standardul ImageNet).
\end{itemize}

\subsection{Hook-ul \texttt{useOnnxClassifier}}

Hook-ul \texttt{useOnnxClassifier} gestionează ciclul de viață al sesiunii ONNX:
\begin{enumerate}
  \item \textbf{Încărcare model} — asincron la montarea componentei;
  \item \textbf{Preprocesare imagine} — decodificare JPEG, redimensionare la $224\times224$,
    normalizare și construirea tensorului;
  \item \textbf{Inferență} — \texttt{session.run(feeds)};
  \item \textbf{Postprocesare} — aplicarea \textit{softmax} și selectarea top-K clase.
\end{enumerate}

\subsection{Preprocesarea imaginilor}

Preprocesarea implementată reproduce exact pipeline-ul standard ImageNet:

\begin{lstlisting}[language=Java, caption={Preprocesarea imaginii pentru ResNet-50}]
function preprocessImage(base64: string): Tensor {
  const buffer = Buffer.from(base64, 'base64');
  const decoded = decodeJpeg(buffer, { useTArray: true });
  const resized = resizeToSquare(decoded, 224);
  const tensorData = new Float32Array(3 * 224 * 224);

  // Normalizare ImageNet
  const mean = [0.485, 0.456, 0.406];
  const std  = [0.229, 0.224, 0.225];

  for (let y = 0; y < 224; y++) {
    for (let x = 0; x < 224; x++) {
      const idx = y * 224 + x;
      const si  = idx * 4; // sursa RGBA
      // Format planar: R, G, B in blocuri separate
      tensorData[idx]               = (resized.data[si]   / 255 - mean[0]) / std[0];
      tensorData[idx + 224 * 224]   = (resized.data[si+1] / 255 - mean[1]) / std[1];
      tensorData[idx + 224 * 224*2] = (resized.data[si+2] / 255 - mean[2]) / std[2];
    }
  }
  return new Tensor('float32', tensorData, [1, 3, 224, 224]);
}
\end{lstlisting}

\subsection{Funcția softmax și selectarea predicțiilor}

Logit-urile de ieșire ale modelului sunt transformate în probabilități prin funcția
softmax numerică stabilă:
\begin{equation}
  \text{softmax}(z_i) = \frac{e^{z_i - \max(\mathbf{z})}}{\sum_{j=1}^{K} e^{z_j - \max(\mathbf{z})}}
  \label{eq:softmax}
\end{equation}

Predicțiile sunt sortate descendent după probabilitate și returnate primele K (implicit $K=3$).

\section{Modelul Maia Chess pentru jocul de șah cu AI}

\subsection{Descriere model Maia Chess}

\textbf{Maia Chess} este un proiect de cercetare open-source al Universității Cornell
(Reid McIlroy-Young et al., 2020) care antrenează rețele neuronale să joace șah imitând
stilul uman la diverse niveluri Elo. Spre deosebire de motoarele tradiționale (Stockfish,
Leela Chess Zero) care maximizează forța de joc, Maia optimizează \textit{acordul cu
mutările umane} la un Elo dat.

Modelul utilizat este \textbf{maia\_rapid.onnx} — un model unic (rapid model) care
acoperă intervalul Elo 1100–1900. Nivelul de joc este controlat prin doi parametri
de intrare suplimentari: \texttt{elo\_self} și \texttt{elo\_oppo}.

\subsection{Arhitectura modelului}

Modelul Maia are \textbf{3 intrări}:
\begin{enumerate}
  \item \texttt{boards} — tensor de tip \texttt{float32} cu forma $[1, 18, 8, 8]$
    reprezentând starea tablei;
  \item \texttt{elo\_self} — tensor de tip \texttt{int64} cu forma $[1]$, categoria Elo
    a modelului (0–10);
  \item \texttt{elo\_oppo} — tensor de tip \texttt{int64} cu forma $[1]$, categoria Elo
    a adversarului.
\end{enumerate}

Și o \textbf{ieșire}:
\begin{itemize}
  \item \texttt{logits\_maia} — tensor de tip \texttt{float32} cu forma $[1, 1880]$,
    reprezentând scorul pentru fiecare mutare din dicționarul oficial de 1880 mutări
    UCI al proiectului Maia.
\end{itemize}

\subsection{Tehnica de oglindire FEN}

O problemă critică în implementare este că modelul Maia a fost antrenat întotdeauna
din perspectiva albului. Prin urmare, când negrul este la mutare, tabla trebuie
\textbf{oglindită} înainte de a fi transmisă modelului:

\begin{enumerate}
  \item Rândurile FEN sunt inversate (rândul 8 devine rândul 1 și invers);
  \item Culorile pieselor sunt interschimbate (litere mari $\leftrightarrow$ litere mici);
  \item Rândul la mutare este interschimbat (w $\leftrightarrow$ b);
  \item Drepturile de rocadă sunt oglindite (K $\leftrightarrow$ k, Q $\leftrightarrow$ q);
  \item Câmpul en passant este oglindit (rândul $r \rightarrow 9-r$).
\end{enumerate}

Dacă board-ul a fost oglindit, mutarea selectată de model trebuie și ea oglindită
înapoi la coordonatele originale înainte de a fi executată.

\subsection{Selectarea mutării optime}

Algoritmul de selecție:
\begin{enumerate}
  \item Se generează lista tuturor mutărilor legale cu \texttt{chess.js};
  \item Fiecare mutare legală este convertită în format UCI (cu oglindire dacă e
    necesar) și căutată în \texttt{all\_moves.json} (1880 intrări);
  \item Logit-ul corespunzător este extras din ieșirea modelului;
  \item Se aplică softmax \textbf{doar} pe mutările legale (nu pe toți cei 1880
    candidați) pentru a preveni dispersia probabilistică;
  \item Se returnează mutarea cu probabilitatea maximă.
\end{enumerate}

\begin{lstlisting}[language=Java, caption={Selecția mutării optime Maia}]
const logits = moveScores.map(ms => ms.logit);
const probs  = softmax(logits); // Softmax DOAR pe mutarile legale!

const movesWithProbs = moveScores.map((ms, i) => ({
  ...ms, prob: probs[i]
}));
movesWithProbs.sort((a, b) => b.prob - a.prob);
return movesWithProbs[0].move; // Mutarea cu probabilitate maxima
\end{lstlisting}

Ambele modele ONNX sunt încărcate asincron la montarea componentei corespunzătoare,

\section{Comportamentul hook-ului de geolocalizare}

Hook-ul \texttt{useGeolocation} (\texttt{src/hooks/useGeolocation.ts}) implementează
un mecanism de rezervă explicit, vizibil în codul sursă: coordonatele GPS sunt
obținute via \texttt{@react-native-community/geolocation}, după care se efectuează
o cerere de reverse geocoding către API-ul Nominatim. Dacă această cerere eșuează,
un bloc \texttt{catch} explicit salvează locația cu doar latitudinea și longitudinea,
fără denumirea localității, asigurând că funcționalitatea de localizare nu se
blocheaza în absența conectivității la internet.

\section{Mecanismul de persistență a datelor}

Persistența cu AsyncStorage este proiectată să fie robustă prin două
mecanisme complementare vizibile în codul sursă:
\begin{itemize}
  \item \textbf{Scriere la fiecare modificare de stare} \textemdash{} un \texttt{useEffect}
    care depend de \texttt{state} apelează asincron \texttt{AsyncStorage.setItem}
    la fiecare acțiune de reducer;
  \item \textbf{Versioning} \textemdash{} cheia de stocare \texttt{retro-arcade-study-state-v1}
    este versionată, permițând migrări viitoare de schemă fără pierderea datelor
    existente. La restaurare (acțiunea \texttt{HYDRATE}), starea salvată este
    îmbinată cu \texttt{initialState} ca baseline, protejând împotriva câmpurilor
    lipsă apărute în versiuni mai noi ale aplicației.
\end{itemize}

\section{Concluzii}

Prezenta lucrare a descris proiectarea și implementarea completă a aplicației mobile
\textbf{FocusQuest}, demonstrând viabilitatea unui produs software complex care combină
multiple domenii: productivitate, gamificare, educație și inteligență artificială
\textit{on-device}.

Principalele realizări tehnice ale proiectului sunt:

\begin{enumerate}
  \item \textbf{Integrarea reușită a ONNX Runtime în React Native} — ambele modele
    (ResNet-50 și Maia Chess) funcționează complet offline pe dispozitive Android,
    cu timpi de răspuns acceptabili pentru interacțiunea utilizatorului.

  \item \textbf{Implementarea corectă a motorului Maia Chess} — inclusiv rezolvarea
    problemelor critice de oglindire FEN, codificarea celor 18 canale de intrare,
    maparea Elo la categorii și selecția mutărilor prin softmax pe mutările legale.

  \item \textbf{Un sistem de gamificare complet și coerent} — cu economie virtuală
    (monede), deblocarea progresivă a conținutului, 34 de realizări, inventar cu 78 de

  \item \textbf{Un modul educațional funcțional} — cu exerciții matematice reale,
    randare matematică cu KaTeX şi sistem de progres persistent.

  \item \textbf{O arhitectură solidă} — bazată pe React Context + useReducer cu
    persistență AsyncStorage, TypeScript strict și navigare tipizată.
\end{enumerate}

Aplicația demonstrează că un singur dezvoltator poate construi, în cadrul unui proiect
